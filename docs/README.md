# Docs
- Start with `../CONTEXT_INDEX.md` to choose what to read.
- Key files:
  - `sprint_status.md` — current sprint tasks/blockers.
  - `architecture.md` — current system snapshot.
  - `AGENTS.md` — global instructions/style/testing.
  - `decisions/` — ADRs (one file per decision; template included).
  - `sprint_archive/` — past sprints.
- Keep files concise; add ADRs for notable decisions.
